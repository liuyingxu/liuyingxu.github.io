Flutter相关简介及环境配置

发表于 2019-10-11 | 分类于 flutter | | 次阅读

Flutter 是 Google推出并开源的移动应用开发框架，主打跨平台、高保真、高性能。开发者可以通过 Dart语言开发 App，一套代码同时运行在 iOS 和 Android平台。 Flutter提供了丰富的组件、接口，开发者可以很快地为 Flutter添加 native扩展。同时 Flutter还使用 Native引擎渲染视图，这无疑能为用户提供良好的体验。

相关技术简介

原生开发简介

iOS：Objective-C或Swift语言调用iOS SDK
Android:Java或Kotlin语言直接调用Android SDK

优势：可访问平台全部功能（GPS、摄像头）；
速度快、性能高、可以实现复杂动画及绘制，整体用户体验好；

缺点：平台特定，开发成本高；不同平台必须维护不同代码，人力成本随之变大；
内容固定，动态化弱，大多数情况下，有新功能更新时只能发版；

纯原生开发的问题：
动态化内容需求增大
业务需求变化快，开发成本变大

跨平台技术简介

1、H5+原生（Cordova、Ionic、微信小程序）webview
通过原生的网页加载控件WebView (Android)或WKWebView（ios）来加载
我们称这种h5+原生的开发模式为混合开发 ，采用混合模式开发的APP我们称之为混合应用或Hybird APP ，
如果一个应用的大多数功能都是H5实现的话，我们称其为Web APP 。
WebView是JavaScript与原生API之间通信的桥梁，
传输协议的工具称之为WebView JavaScript Bridge, 简称 JsBridge（核心）

2、JavaScript开发+原生渲染 （React Native、Weex、快应用）
采用Web开发技术栈，社区庞大、上手快、开发成本相对较低。
原生渲染，性能相比H5提高很多。
动态化较好，支持热更新。

3、自绘UI+原生(QT for mobile、Flutter)
通过在不同平台实现一个统一接口的渲染引擎来绘制UI，而不依赖系统原生控件
所以可以做到不同平台UI的一致性，注意，自绘引擎解决的是UI的跨平台问题，
如果涉及其它系统能力调用，依然要涉及原生开发（调取摄像头等系统设备）

具体跨平台技术介绍
Ionic：angelur常用的组件库（官网：ionicframework.com）
快应用：中国的Android手机厂商联合推出的类似于rn的技术
Weex：阿里的kpi项目（不维护了）vue开发
微信小程序：在webview中渲染的，并非原生渲染，但将来有可能原生渲染

flutter简介

跨平台自绘引擎

跨平台自会引擎，flutter与用于构建移动应用程序的大多数框架不同，因为flutter即不使用WebView，
也不使用操作系统的原生控件，相反，flutter使用自己的高性能渲染引擎来绘制widget

高性能(JIT和AOT)

AOT（Ahead of time）:静态编译的程序在执行前全部被翻译为机器码(提前编译)
JIT（Just-in-time）:解释执行的则是一句一句边翻译边运行(即时编译)

为什么采用Dart开发

1、开发效率高
2、高性能
3、快速内存分配
4、类型安全
5、Dart团队就在隔壁

1、安装Flutter SDK（Mac）
https://flutter.io/docs/development/tools/sdk/archive
