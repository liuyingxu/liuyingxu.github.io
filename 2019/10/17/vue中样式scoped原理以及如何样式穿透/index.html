vue中样式scoped原理以及如何样式穿透

发表于 2019-10-17 | 分类于 vue | | 次阅读

每一个vue文件，由三部分组成分别是template，style，script。在style标签上有一个属性为scoped，当一个style标签拥有scoped属性之后，在这个标签之内的样式（不管是直接写的样式或是引入外部样式文件）都只作用于此vue文件。加此属性的目的就是避免组件之间的样式相互污染，出现不必要的css问题

scoped原理

scoped属性效果通过PostCSS实现，PostCSS是一个用 JavaScript 工具和插件转换 CSS 代码的工具，
添加scoped属性后PostCSS会给当前vue组建中的所有DOM节点添加一个独一无二的动态css属性，并且对你书写的css选择器加上一个对应的后缀，如下图

这样在此选择器下书写的样式就只会作用于当前vue文件的DOM元素，即便其他vue组件有相同类名的元素，样式也不会作用到元素上

scoped样式穿透

scoped解决了样式作用域的问题，但是你有可能会遇到这样的场景：比如我在项目中用到了iview的UI库，但是我用到的某个组件满足不了我的需求，我需要覆盖某个元素的样式，但是加上scoped属性之后父组件的样式将不会渗透到子组件中，就识别不了你书写的iview组件的DOM的css选择器。二者矛盾之下我们可以用特殊的方式使某个样式穿透scoped的作用范围
这种方式能够使scoped 样式中的一个选择器能够作用得“更深”，例如影响子组件
